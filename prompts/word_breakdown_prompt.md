# 逐字拆解 Prompt 設計文件

本文件整理自 [scripts/03_run_api.py](../scripts/03_run_api.py) 中實際使用的 `SYSTEM_PROMPT`，目的是讓面試官或閱讀者不需要翻程式碼，也能理解這個 prompt 的設計邏輯與限制。

程式實際執行時，這份 prompt 是直接寫在 `03_run_api.py` 的 `SYSTEM_PROMPT` 常數裡（並非從本檔案讀取），本文件是該內容的說明版本。

---

## Prompt Purpose

這個 prompt 的角色設定是「日文逐字拆解助手」，任務是把輸入的日文句子轉換成固定格式的學習材料，內容包含：

- 完整保留的日文原句
- 逐詞拆解（單詞、假名、中文意思）
- 整句的自然中文翻譯

設計目的不是單純的機器翻譯，而是希望輸出可以被 `03_run_api.py` 的驗證邏輯檢查，並由 `04_merge_outputs.py` 穩定合併成完整文件。因此這個 prompt 的重點不只是「翻得準」，更著重在**格式穩定**與**內容完整**，避免 LLM 漏句、漏編號，或輸出格式跑掉導致後續合併失敗。

## Output Format

輸出必須嚴格遵守以下結構：

```text
【編號】

日文原句

單詞 （假名 / 中文）

單詞 （假名 / 中文）


中文翻譯：
中文翻譯內容
```

格式重點：

- 每個區塊以 `【編號】` 開頭，後面緊接完整的日文原句。
- 逐字拆解區每個詞語獨立一行，詞語之間需空一行。
- 詞語固定格式為 `單詞 （假名 / 中文）`，括號一律使用全形括號，例如 `（假名 / 中文）`。
- 每個區塊最後必須有 `中文翻譯：` 標籤，後面接自然中文翻譯。

## Strict Rules

以下規則對應原始 prompt 中的「絕對規則」，依主題分組整理，但保留所有限制條件：

**完整性規則**
1. 每個【編號】後面必須完整保留日文原句，不可省略。
2. 不可以省略任何【編號】。
3. 不可以省略任何日文原句。
4. 每一句都必須有「中文翻譯：」。
5. 每一句都必須有逐字拆解。

**詞語格式規則**
6. 每個詞語獨立一行。
7. 每個詞語之間空一行。
8. 詞語格式固定為：`單詞 （假名 / 中文）`。
9. 括號格式固定為：`（假名 / 中文）`。

**標點與結構規則**
10. 不要拆解標點符號。
11. 不要輸出單獨的標點符號行。
12. 不可以把後半句原樣丟進逐字拆解區。
13. 逐字拆解區不能出現沒有括號的長日文句。

**語言限制規則**
14. 中文解釋只能使用繁體中文、必要英文或數字。
15. 中文解釋中禁止混入其他外語文字，例如 Hindi、韓文、阿拉伯文、泰文、西里爾字母、喬治亞字母。

**語氣與助詞規則**
16. 不可以省略句尾表現，例如：んです、なんです、なんですよ、ですよ、ですね、でしょう、だろう、じゃないですか、じゃないですかね、からね、よね。
17. 不要把助詞黏在前一個詞後面（例如「藤原君は」不可以整組拆，必須拆成「藤原君」與「は」兩個詞語）。
18. 常見助詞請分開拆：は、が、を、に、の、と、で、も、から、まで、って、か、よ、ね。

**特殊符號與例外規則**
19. 遇到章節符號「＊」或「◇」時可輸出：＊（＊ / 章節標記）、◇（◇ / 場景切換標記）。
20. 如果原句本身只有標點符號（例如「!」「?」「?!」「「？」」），可以保留原句並給簡單中文翻譯，不需要硬拆成單詞。
21. 短反應句也要拆解，例如：はあ（はあ / 唉；嘆氣聲）、え（え / 咦）、うん（うん / 嗯）、……（…… / 沉默）。

**輸出範圍規則**
22. 不要加入額外說明、總結或評論。

## Design Notes

- **為什麼要用這麼多「絕對規則」？**
  因為輸出會先被 `03_run_api.py` 的驗證邏輯檢查，再由 `04_merge_outputs.py` 合併成最終 Markdown。只要 LLM 漏掉一個編號、漏掉中文翻譯標籤，或是把整句原樣塞進逐字拆解區，驗證階段就會發現問題，避免錯誤內容進入最終學習材料。規則的密度反映的是「長文本批次處理」對輸出穩定性的高要求。

- **助詞獨立拆解（規則 17、18）**
  這是刻意加強的規則，因為日文助詞（は、が、を、に…）在語意上很重要，但容易被模型當成前一個詞的一部分黏在一起輸出。獨立拆解可以讓學習者更清楚句子的文法結構，而不只是背單字。

- **語氣詞不可省略（規則 16）**
  日文句尾語氣（んです、よね、でしょう 等）本身沒有明確的字面翻譯，很容易被模型當作贅字省略。但這些語氣詞對理解說話者的語氣、態度非常關鍵，所以特別列出常見清單，要求模型不能跳過。

- **語言限制（規則 14、15）**
  這條規則是實際使用後才補上的：因為部分 LLM 在生成中文解釋時，偶爾會混入其他語言的字元（例如印地語文字、韓文、西里爾字母），這在教學文件中是明顯的錯誤輸出，因此明確列出禁止清單，並在下游（`03_run_api.py` 的驗證與重試邏輯）搭配格式檢查與自動重試機制。

- **prompt 與程式碼的關係**
  這份文件是 `SYSTEM_PROMPT` 的可讀版本，兩者內容應保持一致。若未來調整 `03_run_api.py` 裡的規則，建議同步更新本文件，避免文件與實際行為不一致。
