# Case Study: Japanese Reading Assistant

## 專案概要

Japanese Reading Assistant 是一個以「用小說學日文」為核心的長文本閱讀輔助工具。

它可以將 EPUB 或 TXT 文本轉換成逐字拆解格式，包含：

- 日文原句
- 詞彙拆解
- 假名
- 中文意思
- 自然中文翻譯

這個專案的目標是降低日文小說的閱讀門檻，讓初級到中級學習者也能透過有趣的原文內容持續學習日文。

---

## 問題背景

我認為日文學習最大的困難之一，不只是文法或單字量，而是如何長期持續下去。

小說、故事和角色對話本來是很適合長期學習的材料，因為它們比教科書例句更有情境，也更容易讓人想繼續讀下去。

但對初級或中級學習者來說，直接閱讀日文小說的門檻很高。

最大的問題是查單字的效率太低。

學習者常常需要一邊讀、一邊查單字、一邊猜助詞、句尾語氣和修飾關係。閱讀過程會被頻繁打斷。即使最後查到了單字，也不一定能理解整句話的結構和語氣。

久了之後，原本有趣的小說也會變成負擔。

因此我想做一個工具，讓學習者可以用更低的成本接觸原文小說，不用一直中斷閱讀去查字典，也能透過逐字拆解理解句子的組成。

---

## 解決方案

Japanese Reading Assistant 會將日文長文本轉換成適合學習者閱讀的逐字拆解教材。

整體流程如下：

```text
EPUB / TXT
↓
文字抽取與清理
↓
日文句子切分
↓
chunk 合併
↓
LLM API 逐字拆解
↓
輸出檢查與重試
↓
最終 Markdown 合併
```

這個專案不是單純把文章丟給 AI 翻譯，而是把長文本處理拆成多個可追蹤、可重複執行的步驟。

---

## 技術設計

### 1. EPUB 文字抽取

`00_extract_epub_text.py` 會讀取 EPUB 內部結構，依照 OPF spine 順序抽取章節文字，並移除 ruby、rt、rb 等振假名標籤。

這可以避免章節順序錯亂，也能讓輸出的 TXT 更適合後續處理。

---

### 2. 日文文本切分

`01_split_paragraphs.py` 會根據日文句尾標點切句，並處理日文對話中常見的引號連接問題。

例如：

```text
「そうだね」「うん」
```

會被拆成：

```text
「そうだね」
「うん」
```

這樣可以避免模型在後續拆解時混淆句子邊界。

---

### 3. 兩階段文本切分策略

這個專案使用兩階段文本切分策略。

第一階段先切成語意完整的句子與小 chunk。  
第二階段再合併成可追蹤的 API 輸入格式。

這樣可以避免：

- token 超限
- 句子被切斷
- API 呼叫過於零碎
- 中途失敗後難以恢復
- 錯誤來源難以追蹤

---

### 4. API 批次處理

`03_run_api.py` 會逐一處理每個 FILE 區塊，並將結果保存成獨立檔案。

每個輸出都會保留句子編號範圍，例如：

```text
001_1-8_翻譯.txt
002_9-16_翻譯.txt
```

這讓長文本處理可以中斷後繼續，也能避免單次失敗影響整本內容。

---

### 5. 輸出檢查與重試

API 輸出後，程式會檢查格式是否符合要求。

檢查項目包含：

- 是否漏掉句子編號
- 是否缺少日文原句
- 是否缺少逐字拆解
- 是否缺少中文翻譯
- 是否出現重複編號
- 是否出現範圍外編號
- 是否把整句原文誤放進逐字拆解區
- 是否出現括號格式錯誤

如果發現嚴重問題，程式會自動重試。  
如果重試後仍然失敗，問題輸出會被另外保存，避免錯誤內容進入最終檔案。

---

## 專案成果

目前專案已經可以完成以下流程：

```text
TXT / EPUB
↓
切分
↓
合併 API 輸入
↓
API 批次處理
↓
輸出檢查
↓
合併最終 Markdown
```

範例輸出可以查看：

```text
sample/final_output.md
```
